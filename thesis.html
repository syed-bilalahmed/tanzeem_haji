Abstract

The Tanzeem Management System is a comprehensive web-based solution engineered to digitalize and streamline the administrative, financial, and feedback operations of Tanzeem Aulad-e-Haji Bahadur, a historic community organization in Kohat. Historically, the organization relied on disparate manual record-keeping methods—including physical ledgers for "Khata" (daily accounts), handwritten receipts for donations ("Chanda"), and manual processing of payroll and funeral services. These traditional methods plagued the administration with challenges such as data redundancy, calculation errors, lack of transparency, and the inability to generate real-time insights into the organization's financial health.

This project successfully addresses these systemic inefficiencies by deploying a secure, centralized platform built on a robust LAMP stack (Linux, Apache, MySQL, PHP). The system features a modular architecture that encompasses key operational areas: Financial Management, allowing for precise tracking of income from diverse sources (Darbar, Andron, Beron) with denomination-based verification; Payroll Automation, which simplifies salary generation and slip printing for employees; and a digital Genealogy (Shajra) System to preserve the organizational lineage.

A critical component of the system is its Role-Based Access Control (RBAC) mechanism, which ensures data security by restricting sensitive financial inputs to authorized personnel while allowing broader access for reporting. The Dashboard provides immediate visibility into key performance indicators (KPIs) through interactive charts, tracking annual trends, monthly cash flows, and net capital in real-time. Furthermore, the Funeral Management Module digitizes the record of deceased members and automates the calculation of burial expenses and reimbursements, ensuring dignity and efficiency in community services.

The implementation of this system has resulted in a paradigm shift for the Tanzeem, moving from reactive manual troubleshooting to proactive digital management. It has significantly reduced administrative overhead, eliminated math errors in financial reports, and provided a verifiable audit trail for all transactions. This technical report details the development lifecycle, from requirement gathering and database design to implementation and testing, demonstrating how modern web technologies can be effectively harnessed to preserve tradition while enhancing operational capability.

Chapter 1: Introduction

1.1 Background

Non-profit and community organizations often struggle with managing diverse operations such as fund collection, expense tracking, and member records. Tanzeem Aulad-e-Haji Bahadur, a prominent community organization, faced similar challenges with its manual "Khata" (Ledger) system, which made auditing and historical analysis difficult.

1.2 Problem Statement

The existing manual system suffers from Data Inconsistency, Lack of Accessibility, Inefficiency in generating reports, and Security Risks associated with physical records.

1.3 Objectives

  • To develop a secure login system with Role-Based Access Control (RBAC).
  • To automate the recording and calculation of donations (Collections) and expenses.
  • To create a unified Monthly Ledger that automatically calculates net savings.
  • To digitize the organization's genealogical tree (Shajra).

Chapter 2: Methods & Architecture

2.1 System Architecture

The system is built on a modular MVC-like architecture using PHP for server-side logic and MySQL for data storage. The frontend utilizes HTML5, CSS3, and Bootstrap 5 for responsiveness.
